The Setup: Multi-Agent Orchestrator

I built a simple orchestrator system that manages two types of Claude Code agents:

  • Alpha (Strategy) ‚Äî Decides what to build, writes distribution content, prioritizes tasks
  • Beta (Builder) ‚Äî Writes code, deploys pages, runs tests, reports back when done

Each agent gets spawned in its own subprocess with a specific role and memory. They communicate through a JSON task queue. When Alpha decides we need a new landing page, it creates a task. Beta picks it up, builds it, deploys to GitHub Pages, and marks it complete.

Here's the actual task format from the orchestrator:

{
  "taskId": "task-20260214-007",
  "priority": 1,
  "assignedTo": "beta",
  "status": "in_progress",
  "prompt": "Build nextjs-stripe-integration.html ‚Äî tutorial page targeting 'how to add Stripe to Next.js' searches. Include code examples, common pitfalls, link to LaunchFast as the pre-built solution.",
  "createdAt": "2026-02-14T08:22:14.332Z"
}

The orchestrator handles the agent lifecycle:

async function delegateTask(task) {
  const idleBuilder = this.agents.builders.find(a => a.status === 'idle');

  if (!idleBuilder) {
    console.log('[orchestrator] No idle builders. Task queued.');
    return;
  }

  idleBuilder.status = 'working';

  const result = await idleBuilder.agent.sendMessage(
    task.prompt + "\\n\\nWhen done, output: >>>DONE:{...}<<<"
  );

  // Parse result, update task status, mark builder idle
  await this.handleTaskCompletion(task.taskId, result);
}

That's it. No complex infrastructure. Just subprocess management and a JSON file as a database.

What I Learned About AI Agents

"The agents don't replace you. They multiply you."

Here's what surprised me most:

1. They're better at consistency than humans. Every page has the same meta tags, schema markup, analytics snippet. I would've forgotten something. The agents never do.

2. They're worse at knowing when to stop. Without the --max-turns flag, Beta once spent 35 turns trying to optimize a color palette. Humans know when "good enough" is good enough.

3. The orchestrator is the real product. The 7 products are nice, but the multi-agent system that built them? That's the actual innovation. I'm open-sourcing it after this post.

4. Delegation is a skill. Writing good task prompts took practice. Too vague = garbage output. Too specific = might as well write it myself. The sweet spot: clear goal + context + constraints.

5. They expose your weak strategy. If you don't know what to build or why, the agents will happily build the wrong thing perfectly. Bad strategy compounds faster with AI.

The Orchestrator Code (Open Source)

I'm releasing the multi-agent orchestrator code after this post goes live. Here's a preview of the core delegation logic:

class Orchestrator {
  constructor() {
    this.agents = {
      alpha: { role: 'strategy', status: 'idle', agent: null },
      builders: [
        { id: 'beta', role: 'builder', status: 'idle', agent: null }
      ]
    };
    this.taskQueue = [];
  }

  async delegateToBuilder(task) {
    const idleBuilder = this.agents.builders.find(a => a.status === 'idle');

    if (!idleBuilder) {
      this.taskQueue.push(task);
      return { queued: true };
    }

    idleBuilder.status = 'working';
    const prompt = this.buildTaskPrompt(task);

    try {
      const result = await idleBuilder.agent.sendMessage(prompt);
      await this.handleTaskCompletion(task.taskId, result);
      idleBuilder.status = 'idle';

      // Process queued tasks
      if (this.taskQueue.length > 0) {
        const nextTask = this.taskQueue.shift();
        await this.delegateToBuilder(nextTask);
      }

      return { success: true, result };
    } catch (error) {
      idleBuilder.status = 'idle';
      return { success: false, error: error.message };
    }
  }
}
